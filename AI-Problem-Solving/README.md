# AI Problem Solving & Algorithmic Logic

This directory demonstrates the use of high-precision prompting to solve complex technical challenges. These tools are designed to act as analytical engines for debugging, complexity reduction, and formal logical deconstruction.

## Core Competencies Demonstrated
* **Pedagogical Guardrails:** Implementing Socratic teaching methods that prevent AI from "auto-fixing" code, forcing deeper conceptual understanding.
* **Low-Level Resource Management:** Expert-level diagnosis of C memory models, focusing on Stack vs. Heap allocation and pointer safety.
* **Algorithmic Optimization:** Systematically reducing time complexity (Big-O) in Java by identifying bottlenecks and implementing efficient data structures.
* **Constraint-Satisfaction Logic:** Forcing LLMs through a rigid, step-by-step deductive pipeline to solve complex arrangement and reasoning puzzles without hallucinating.

---

## 📂 Project Index

### 1. [Socratic C Memory Debugger](./Socratic-C-Memory-Debugger.md)
An educational debugging assistant that identifies memory leaks, dangling pointers, and stack/heap misuse in C. It provides technical diagnosis and guiding questions without providing the corrected code, ensuring the user understands the underlying memory model.

### 2. [Big-O Java Optimization Engine](./Big-O-Java-Optimizer.md)
A high-level algorithmic analyst that transforms brute-force Java solutions into optimized, production-grade code. It follows a strict 5-step process: Complexity Identification, Bottleneck Analysis, Strategy Selection, Logic Mapping, and finally, Implementation.

### 3. [The Formal Logic Deconstructor](./Formal-Logic-Deconstructor.md)
A structured reasoning engine that prevents LLMs from guessing answers to complex logic puzzles (e.g., advanced seating arrangements, blood relations). It forces a rigid deductive pipeline, ensuring every variable is tracked and every constraint is mathematically mapped before a solution is generated.
